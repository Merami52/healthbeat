# HealthBeat  
Учебный проект, демонстрирующий основы отказоустойчивости, мониторинга и устранения сбоев.

## Почему этот проект  
Сервисы сегодня должны работать надёжно: быстро восстанавливаться после сбоев, быть под наблюдением, обеспечивать видимость метрик и состояния здоровья.  
HealthBeat создан как мини-лаборатория, где можно отработать навыки: запустить сервис, включить мониторинг, симулировать отказ и посмотреть, как всё ведёт себя в контейнерах.

## Основные возможности  
- REST-сервис на FastAPI, реализующий основные эндпоинты  
- Контейнеризация с Docker и docker-compose  
- Мониторинг с Prometheus: сбор метрик сервиса  
- Эндпоинты «health-check», симуляция отказов и сбоя  
- (При наличии: логирование, тесты, масштабируемость)  

## Архитектура и используемые технологии  
| Компонент | Технология |
|-----------|-----------|
| Язык разработки | Python |
| Web-фреймворк | FastAPI |
| Мониторинг | Prometheus |
| Контейнеризация | Docker, docker-compose |
| Зависимости | См. `requirements.txt` |
